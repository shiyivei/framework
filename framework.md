# 1 架构思维vs程序思维

程序 = 翻译 + 逻辑 + 实现

架构 = 判断 + 取舍 + 创新

架构设计的本质是“判断+取舍”，程序设计的本质是“逻辑+实现”

# 2 架构师的定位

业务和技术之间的桥梁

# 3 架构师能力模型

技术（技术深度 + 技术宽度 + 技术广度）

业务 （级别越高 + 业务理解要求越高）

管理 （团队管理 + 业务管理）

# 4 发展路线

## 4.1 高级工程师

掌握基础原理：JVM、开源软件，TCP，HTTP

学习套路：分库分表、缓存、SOLID、设计模式、MVP等

## 4.2 技术专家

熟悉核心源码：成熟的开源软件，例如Redis，Nginx，Netty

链式学习法与比较学习法，技术宽度和深度

## 4.3 初级架构师

成为领域专家

业界交流：参加技术大会，关注大厂技术

# 5 面试与面评

判断力：业务复杂性问题

拆解能力：设计合适的架构

取舍能力：决策取舍逻辑是否正确
